# Sample Audit Reports

Every report in this folder is **synthetic**: the projects, names, paths,
and findings are invented to show what a good blindspot-audit output looks
like. No real project data is included.

What each sample demonstrates:

- `web-app-launch.md` (English) - a solo engineer's side project about to
  go public: owner-inverse weighting (legal/ops findings for an engineer),
  the regulated-domain hedge, and a peer expectation gap phrased as a
  decision.
- `indie-game-steam.md` (English) - a first Steam release: long-lead-time
  platform mechanics, source-tier handling in the fresh-eyes scan, and
  stage-appropriate scoping.
- `research-analysis.md` (English) - a data-analysis-to-paper project:
  reproducibility and provenance findings, and what "checked and well
  covered" looks like when the owner is strong in the domain.
- `web-novel-serialization.ko.md` (Korean) - 연재 직전 웹소설 프로젝트:
  보고서 언어는 소유자의 언어를 따르고 ID/status만 영어로 유지하는 규칙,
  한국어 numbered awareness check의 실제 모양.
- `weak-vs-strong.md` (English) - three findings written twice: the
  checklist-nag version that fails the ground rules, and the
  evidence-based version that passes. Read this one first.

All samples follow `references/report-template.md`. They are deliberately
compact - a real deep audit may carry more evidence detail, but the shape
should look like this.
